# Adaptive Learned Index

A high-performance C++17 learned index built on piecewise linear segmentation, featuring calibrated adaptive search bounds, an ensemble distribution-drift detector, and a buffered insert layer. Benchmarked against B-Tree, ALEX, PGM, and RMI on three real-world SOSD datasets at 100K and 1M key scales.

## How NLI Works

NLI is a two-level piecewise linear learned index with four co-designed components.

### Two-Level Segmented Structure

Keys are indexed through two levels of piecewise linear segments (PGM-style). The root level routes a lookup to a leaf segment; the leaf segment's linear model predicts the position in the key array. A bounded search window (controlled by CAAB) finds the exact key.

```
Query Key
    │
    ▼
Root Segments  ──(SRLM-2: 2 sub-models, tight BREC window)──►  Leaf Segment
    │
    ▼
Leaf predict  ──(EBECS bias correction)──►  ECS window search  ──►  Key found
    │
    ▼ (miss)
BIB — Ghost Insert Buffer  (hot ring 32 entries + warm sorted buffer 4096 entries)
```

### Core Components

**CAAB — Calibrated Adaptive Autoscaled Bounds**
The search window around the predicted position is derived from the per-segment P99 error measured at build time, not a global fixed bound. This keeps windows tight on well-modelled segments and wider where the model is less accurate.

**EIDD-S — Ensemble Incremental Drift Detection (sampled)**
Four statistical detectors run in parallel, evaluated every 128 queries to keep overhead below 6 %. A drift alarm fires when at least 2 of 4 detectors agree:

| Detector | Trigger |
|---|---|
| EWMA | Exponentially weighted mean error exceeds 3× stable baseline (α = 0.005) |
| PSI | Population Stability Index across 16 equal-width buckets exceeds 0.20 |
| KS | Two-sample Kolmogorov-Smirnov statistic on a rolling 2000-query window (α = 0.01) |
| AE | TinyAE autoencoder (16→4→16, 84 params) reconstruction error on rolling error windows |

**ETIR — Efficient Targeted Incremental Repair**
On confirmed drift: flush the BIB → refit OLS segments → recalibrate CAAB. Completes in 5–15 ms. EWMA state is preserved (warm restart) to skip the 500-query re-warmup after each repair.

**BIB — Buffered Insert Buffer**
A dual-stage insert buffer: a hot ring of 32 entries (O(1) insert) backed by a sorted warm buffer of up to 4,096 entries (O(log k) lookup). Amortises structural rebuilds across many inserts.

### The 12 Query-Path Optimisations (v7.0)

| # | Name | Effect |
|---|---|---|
| 1 | ILC — In-Leaf CAR | `bias` + `seg_eps` embedded in the 32-byte `LeafSeg` struct; removes 2 cache misses per query |
| 2 | BTSC — Build-Time Segment Calibration | Per-segment `seg_eps` and signed `bias` computed from all training keys at build time; cold-start accurate |
| 3 | EBECS — Error-Biased ECS | Shifts the search centre by the learned `bias`; raises exact-match rate from ~40 % to ~65 % |
| 4 | SRLM-2 — Two Sub-Model Root | Root keys split at median; two independent linear models halve root residual |
| 5 | BREC — Build-Time Root Epsilon Calibration | Measures actual max leaf prediction error post-build; sets tight root search window |
| 6 | SPP+ — Enhanced Staged Prefetch Pipeline | Two prefetch pairs issued at Steps 2 and 4 to overlap L2/L3 miss latencies |
| 7 | WSEWMA — Warm-Start EWMA | EWMA state saved before repair and restored immediately after; no re-warmup blind period |
| 8 | LDZED — LDZ Exponential Decay | Segment error counters halved on partial repair instead of zeroed; prevents oscillation |
| 9 | GIDB — Ghost Insert Dual Buffer | Hot ring (O(1)) + warm sorted buffer (O(log k)); 97 % of inserts bypass the sorted path |
| 10 | RCO — Repair Cooldown | Alarms suppressed for 2,000 queries after any repair; prevents repair storms on gradual drift |
| 11 | AFR — Accelerated Full Repair | Caches last-used epsilon; warm-starts rebuild from it, skipping 1–3 redundant `pgm_build()` calls |
| 12 | MLH — Monotone Lookup Hint API | Public `lookup_hint(key, seg_hint, out)` for scan iterators to bypass root routing |

## Dataset Setup

Place the three SOSD binary dataset files in `sosd_data/` before running:

```
sosd_data/
├── books_200M_uint64
├── fb_200M_uint64
└── wiki_ts_200M_uint64
```

**Format**: 8-byte little-endian uint64 key count followed by N × 8-byte sorted uint64 keys.

Download from the [SOSD repository](https://github.com/learnedsystems/SOSD). The compressed `.zst` originals are also kept in `sosd_data/` for reference.

Verify datasets before running:

```bash
python3 scripts/validate_datasets.py sosd_data
```

## Build

### Linux / macOS / Git Bash

```bash
# Release build (default)
./build.sh

# Debug build — enables AddressSanitizer + UBSan
./build.sh debug

# Clean build directory
./build.sh clean
```

### CMake directly

```bash
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . -j$(nproc)
```

### Windows (PowerShell)

```powershell
# Requires MinGW g++ in PATH
# (MSYS2: pacman -S mingw-w64-x86_64-gcc)
.\run_all.ps1 sosd_data results
```

Three binaries are produced in `build/`:

| Binary | Purpose |
|---|---|
| `nli_benchmark` | Main read + insert benchmark |
| `nli_drift` | Drift detection & repair benchmark |
| `nli_ablation` | Component ablation study |

## Running the Full Benchmark

```bash
# Interactive — prompts for key size
./run_all.sh sosd_data results

# Non-interactive — 100K and 1M keys
./run_all.sh sosd_data results 100000 1000000

# Quick test — 100K only (~1–2 min)
./run_all.sh sosd_data results 100000
```

The pipeline runs in five steps:

1. **Compile** — builds all three binaries
2. **Main benchmark** — B-Tree / ALEX / PGM / RMI / NLI-Full / NLI-Linear / NLI-NoDrift
3. **Drift benchmark** — 5 drift scenarios × 3 datasets × selected scales
4. **Ablation study** — NLI-Full vs 7 component-ablated variants
5. **Figures & report** — calls `generate_figures.py`, writes figures to `figures/`

### Run individual benchmarks

```bash
./build/nli_benchmark sosd_data results 100000 1000000
./build/nli_drift     sosd_data results 1000000
./build/nli_ablation  sosd_data results 100000 1000000
```

### Regenerate figures from existing CSVs

```bash
python3 generate_figures.py results figures
```

## NLI Variants

| Variant | What is disabled |
|---|---|
| NLI-Full | Nothing — complete system |
| NLI-Linear | MLP / residual correction off |
| NLI-NoDrift | Drift detection off |
| NLI-NoRepair | Detection on, repair suppressed |
| NLI-NoPW | Piecewise model → single linear model |
| NLI-NoSRLM2 | Two sub-model root → single root model |
| NLI-NoBTSC | Build-time calibration off |
| NLI-NoRCO | Repair cooldown off |

## Build Flags

Always use these flags to run the official third-party baseline implementations:

```
-DUSE_REAL_ALEX -DUSE_REAL_PGM -DALEX_USE_LZCNT=0
```

Without these flags the code compiles its own faithful reference implementations of ALEX and PGM, which is useful for portability but may show different absolute numbers.
